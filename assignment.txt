Task 1 — Basic Query
Write a SQL query (pseudocode is fine) to find all active devices that have had zero usage in the last 7 days.
SELECT d.device_id, d.org_id
FROM devices d
WHERE d.status = 'active'
  AND d.device_id NOT IN (
    SELECT u.device_id
    FROM usage u
    WHERE u.date >= CURRENT_DATE - INTERVAL 7 DAY
      AND u.mb_used > 0
  );

Task 2 — Aggregation
Write a SQL query to return total MB used per org in the last full calendar month.
SELECT d.org_id, SUM(u.mb_used) AS total_mb
FROM devices d
JOIN usage u ON d.device_id = u.device_id
WHERE u.date >= DATE_TRUNC('month', CURRENT_DATE - INTERVAL '1 month')
  AND u.date < DATE_TRUNC('month', CURRENT_DATE)
GROUP BY d.org_id;

Section C — API Troubleshooting Scenario
A customer sends this POST request to activate SIMs:
POST /v1/devices/activate
Headers:
  Authorization: Bearer 12345
  Content-Type: application/json

Body:
{
  "iccid": "893810000012345678",
  "plan": "basic"
}

They receive this JSON response:
{
  "error": "invalid_request",
  "message": "Missing required field: 'organization_id'"
}

They insist:
“Your API is broken. We already provided the ICCID and plan. Please fix.”
Your Task
What is wrong with the request?
The body is missing a key-value pair expected by the endpoint  /v1/devices/activate.
Rewrite the corrected API request.
POST /v1/devices/activate
Headers:
  Authorization: Bearer 12345
  Content-Type: application/json

Body:
{
  "iccid": "893810000012345678",
  "plan": "basic",
  “organization_id”: {org_id found on dashboard}
}
Explain—briefly—how you’d communicate this to the customer in a friendly, clear way.
Hi [name],

Thanks for sending that over; I can see what's happening here. The API isn't broken; the issue is that the /v1/devices/activate endpoint requires three fields in the request body: iccid, plan, and organization_id. Your request includes the first two, but the organization_id field is missing, which is why you're seeing the "invalid_request" error.
Here's the corrected request:

POST /v1/devices/activate
Headers:
  Authorization: Bearer 12345
  Content-Type: application/json

Body:
{
  "iccid": "893810000012345678",
  "plan": "basic",
  "organization_id": YOUR_ORG_ID
}

You can find your organization_id in the dashboard under Settings → Organization. Once you add that field, the activation should succeed. Let me know if you run into any other issues.

Thanks,

Tom

import json
events_json = {
    "events": [
        {"ts": "2025-01-03T10:00:00Z", "type": "attach", "result": "fail"},
        {"ts": "2025-01-03T10:01:00Z", "type": "attach", "result": "success"},
        {"ts": "2025-01-03T10:02:00Z", "type": "data", "mb": 1.2}
    ]
}

# Find the first attach event with result="success"
for event in events_json["events"]:
    if event.get("type") == "attach" and event.get("result") == "success":
        print(event["ts"])
        break
